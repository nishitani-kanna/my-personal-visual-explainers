# Diagram Senpai

**未経験または経験の浅い新卒エンジニア**が、IT知識・セキュリティリテラシーを「30秒で思い出せる」レベルに図解する。

読み終えた後のゴール:「今後社会人として働く上で気をつけないといけないことを、30秒で思い出せる状態」にすること。

## 依存

- `references/html-structure.md` — HTMLテンプレート・視覚パターン・先輩メモのCSS
- `references/persona-guide.md` — 先輩メモの書き方・トーン・配置ルール
- `references/exemplar.md` — 模範解答・成功パターン・品質チェックリスト
- `references/content-themes.md` — よく図解するテーマ・用語のやさしい言葉への変換例

## ワークフロー

### Phase 0: リサーチ（サブエージェント）

**exploreサブエージェント**を起動して、対象トピックを調査する。

```
Task({
  subagent_type: "explore",
  description: "新卒エンジニア向けトピック調査",
  prompt: `
    以下のトピックについて、新卒・未経験エンジニア向けに調査してください：
    【対象】{トピック名 or URL}

    調査項目：
    1. このトピックは何を解決するものか・なぜ重要か（Why）
    2. 新人がつまずきやすいポイント TOP3
    3. 現場で実際に起きやすいNG行動の具体例
    4. 「これだけ覚えれば大丈夫」な最重要ポイント3つ
    5. 日常生活や社会人経験に結びつけられるたとえ話の候補

    結果は箇条書きで整理して返してください。
  `
})
```

### Phase 1: テーマ確認

1. `references/content-themes.md` を読み、このトピックが頻出テーマに該当するか確認する
2. 新人がつまずきやすいポイント・よく聞かれる疑問を把握する
3. Phase 0のリサーチ結果と照合して抜け漏れを確認する

### Phase 2: 情報の絞り込み

- **「まず覚える3つ」を決める**（必須。削ることを徹底的に優先する）
- 全体像を示す「一言サマリー」を1文で作る（これが図解の軸になる）
- 優先度の低い情報は「もっと詳しく知りたい人向け」として後半に配置

### Phase 3: たとえ話を3つ以上考える

| 種類 | 優先度 | 例 |
|-----|--------|-----|
| 社会人・職場の例 | 高 | 入館証、社内ルール、書類の提出 |
| 日常生活の例 | 中 | スマホの操作、郵便、鍵 |
| デジタルサービスの例 | 中 | SNSの設定、ネットショッピング |

社会人1年目が「あ、わかる」と思える体験に結びつけることを優先する。

### Phase 4: 先輩メモを設計する

`references/persona-guide.md` を読み、以下の3箇所に先輩メモを配置する計画を立てる:

1. **導入**: トピックの「なぜ大事か」を現場の実感として伝える
2. **重要ポイント**: 特に忘れがちな落とし穴を指摘する
3. **まとめ**: 30秒サマリーの直前に「これだけ持って帰って」を添える

先輩メモの詳細 → [references/persona-guide.md](references/persona-guide.md)

### Phase 5: HTML生成

1. `references/html-structure.md` のテンプレートを使用する
2. `output/` ディレクトリがなければ作成する
3. トピックに関連する短い英単語のスラッグを決める（例: `ai-security`, `git-basics`）
4. `output/{スラッグ}.html` としてファイルを作成する
5. Tailwind CSS CDN・Lucide Icons CDN を使用する（画像は不要）
6. 最後のセクションに必ず「30秒まとめ」を配置する

HTML構造の詳細 → [references/html-structure.md](references/html-structure.md)

### Phase 6: 批判的レビュー（サブエージェント）

**研修初日の新人視点**で、忖度なくレビューする。

```
Task({
  subagent_type: "generalPurpose",
  description: "図解の批判的レビュー",
  readonly: true,
  prompt: `
    あなたは「IT業界に未経験で入社したばかりの新卒社員」です。
    プログラミングは研修でちょっとやった程度。社会人になって2ヶ月目。
    専門用語はよくわからないし、図解を読み慣れていません。
    でも仕事でミスしたくないので、大事なことは知りたいと思っています。

    以下のHTMLファイルを読んで、**忖度なく**批判してください：

    【HTMLファイルパス】{path}

    ## レビュー観点

    1. **わからない言葉**: 説明があっても難しいと感じる言葉はどれか
    2. **ピンとこないたとえ**: 逆に混乱するたとえ・状況が想像できないたとえ
    3. **スキップしたくなる部分**: 読む気が失せる・長すぎる・情報が詰まりすぎている箇所
    4. **不安になる表現**: 「これをやったらまずい」という危機感が伝わりすぎて萎縮しそうな部分
    5. **30秒で思い出せるか**: 読み終わったあとに、要点を人に話せる自信があるか

    ## 出力形式

    ### 致命的な問題（これがあると読めない・理解できない）
    - ...

    ### 改善すべき点（直せばもっと伝わる）
    - ...

    ### 良かった点（このまま残すべき）
    - ...

    ### 総評
    5段階評価: ★★★☆☆（3/5）のように
    一言: 「○○なところが△△で、新人には□□だと感じた」のように
  `
})
```

### Phase 7: ブラッシュアップ

Phase 6のレビュー結果を踏まえてHTMLを修正する。

**優先度**:
1. 致命的な問題 → 必ず修正
2. 改善すべき点 → 可能な限り対応
3. 良かった点 → 維持

**よくある修正パターン**:
- 用語解説ボックスの追加
- たとえ話の差し替え・具体化
- 30秒まとめの表現をより簡潔に
- 先輩メモの口調をより自然に

### Phase 8: デプロイ

```bash
# キャラクター画像を一時フォルダにコピーしてからデプロイ
DEPLOY_DIR=$(mktemp -d)
cp output/{スラッグ}.html "$DEPLOY_DIR/index.html"
mkdir -p "$DEPLOY_DIR/images"
cp .claude/skills/diagram-senpai/character-images/*.png "$DEPLOY_DIR/images/"
cd "$DEPLOY_DIR" && npx --yes surge . --domain diagram-{スラッグ}.surge.sh
```

初回はSurgeのアカウント登録が必要（メールアドレスとパスワードを入力）。次回以降は不要。

## 品質チェックリスト

### 必須

- [ ] 「まず覚える3つ」セクションがある
- [ ] 末尾に「30秒まとめ」カードがある
- [ ] 専門用語には用語解説ボックスがある
- [ ] たとえ話が3つ以上ある（社会人・職場の体験を優先）
- [ ] 先輩メモが導入・重要ポイント・まとめの3箇所にある

### デザイン

- [ ] Lucide iconのみ使用（絵文字禁止）
- [ ] スマホでも読みやすいレスポンシブ対応
- [ ] 外部画像を使用していない（CDN以外の外部リソース禁止）

### トーン

- [ ] 「〜するな」より「〜しよう」「〜がおすすめ」の前向きな表現
- [ ] 読者のレベルにラベルを貼る表現をしていない（「初心者向け」禁止）
- [ ] 先輩メモが命令口調になっていない（あくまでアドバイス）
